Architecture de l'application

L'application respecte une architecture répartie permettant de contourner les restrictions CORS des navigateurs :

  • Front-end (Client) : Développé en HTML/CSS et TypeScript. Utilisation de Leaflet.js pour la cartographie. Transpilé avec esbuild.
  • Back-end (Proxy Java) : Un serveur HTTP personnalisé utilisant com.sun.net.httpserver. Il agit comme un pont pour interroger les données distantes.
  • APIs Distantes : JCDecaux (Norme GBFS pour les vélos) et API de la Métropole du Grand Nancy (Incidents).

Ce qui a été fait

  • Mise en place de la carte interactive Leaflet centrée sur Nancy.
  • Récupération et fusion des données GBFS (Informations et Statuts) pour les stations de vélos.
  • Création d'un Proxy Java pour contourner le blocage CORS de l'API des incidents de voirie.
  • Création d'icônes dynamiques (vert/rouge pour les vélos selon la disponibilité).
  • Séparation du code TypeScript en modules (Data, Map, App).

Manuel d'utilisation

  1. Lancer le serveur Proxy Java (Main.java) sur le port 8080.
  2. Ouvrir le projet Front-end avec un serveur local (ex: Live Server sur VS Code).
  3. Dans la barre d'outils de la carte, vérifier que l'URL du proxy pointe bien vers http://localhost:8080.
  4. Cliquer sur "Recharger" pour synchroniser les données en temps réel.
